Problem Statement

Mental health disorders like anxiety and depression are a pressing global issue, yet a significant number of cases go undiagnosed. Traditional screening methods, such as clinical questionnaires, can be intrusive, time-consuming, and may suffer from response bias.

This leads to a critical gap in early detection. The core problem is the need for a more engaging, implicit, and accessible screening method that can identify potential mental health risks without relying on direct self-reporting, thereby encouraging wider participation and more honest behavioral data.

Objective

The primary objective is to design, develop, and evaluate a novel system for predicting anxiety and depression scores through gamified behavioral data collection.

Specific goals include:

  1. Develop an interactive, multi-game web application ("Mindful Play") that implicitly captures behavioral metrics.
  2. Create a robust data pipeline for collecting and storing gameplay data in Firebase.
  3. Build and train an MLP model to predict mental health scores from the collected gameplay data.
  4. Provide a user-friendly data export function (CSV) for analysis in Google Colab.

Literature Review

Our work builds upon a strong foundation of existing research. The core inspiration is the study by **Vitória et al. (2024)** [1], which successfully used an MLP model to predict anxiety and depression from questionnaire data. Our project innovates by replacing direct questioning with implicit data gathered from gameplay.

The validity of using gameplay data is supported by multiple studies. For instance, **Al-Saggaf et al. (2024)** [2] demonstrated over 93% accuracy in detecting depression and anxiety markers by analyzing game logs. This confirms that behavioral patterns during play can be powerful predictors. Furthermore, a meta-analysis by **Fleming et al. (2017)** [3] showed that "serious games" can have significant positive effects on mental health, validating gamification as an effective methodology.

Data Collection Pipeline

To make accurate predictions, we need clean, well-structured data. Our pipeline is designed to transform every user interaction into a format ready for machine learning.

1. Raw Interaction

Every click, choice, and reaction time is captured as a raw event in the HTML/JavaScript frontend.

2. Session Structuring

Events are grouped into a user session and stored in Firebase Firestore for persistence.

3. CSV Conversion

The structured session data is flattened and converted into a downloadable CSV file.

4. ML-Ready Data

The final CSV is formatted and ready for feature engineering in Google Colab.

Module 1: UI & Gameplay

The user-facing web application, built with HTML, CSS, and native JavaScript. This module's primary role is to provide an engaging experience through a suite of cognitive games. It handles all visual rendering, manages game state (scores, timers, accuracy), and captures detailed behavioral metrics—like decision paths in narrative games, time-to-completion for puzzles, and hit/miss ratios—which serve as the raw data for our analysis.

Module 2: Data Persistence

This module acts as the backend logic within the frontend. It structures the raw event data into a cohesive, session-based NoSQL format. Using the Firebase SDK, it sends this structured data to a Firestore database. This ensures every game session is securely and persistently stored in the cloud, making the data accessible for later analysis and export. The real-time nature of Firestore also opens possibilities for future live features.

Module 3: AI Integration

This module enhances gameplay by connecting to the Google Gemini API. Its main function is to facilitate dynamic content generation. For instance, in the "Dream Weaver" game, the module sends the user's narrative choices to the Gemini API, which then returns a unique, contextually relevant continuation of the story. This creates a more personalized and unpredictable experience, increasing user engagement.

Module 4: Data Export & Analysis

The crucial bridge between data collection and machine learning. This module contains the client-side JavaScript logic that allows the user to download their gameplay data. It fetches the session data from Firebase, "flattens" the nested JSON structure into a two-dimensional format suitable for a spreadsheet, and converts it into a universally compatible CSV file. This exported file is the final, clean dataset ready for analysis in Google Colab.
